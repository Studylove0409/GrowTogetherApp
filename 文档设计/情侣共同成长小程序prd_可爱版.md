# 《一起进步呀》微信小程序 PRD 文档

> 产品类型：情侣共同成长 / 计划监督 / 每日打卡小程序  
> 文档版本：V1.0  
> 目标版本：MVP 第一版  
> 产品风格：可爱、温暖、轻监督、有陪伴感  
> 适用对象：情侣、亲密关系中的两个人、互相督促成长的搭子  

文档职责：

- `PRD.md` 负责产品功能、页面结构、业务规则、开发优先级。
- `DESIGN.md` 负责视觉风格、颜色、字体、间距、圆角、按钮、卡片与组件规范。
- 当两份文档冲突时，功能结构、页面数量、业务规则、开发优先级以 PRD 为准；视觉样式与组件表现以 DESIGN 为准。

## 1. 产品概述

### 1.1 产品名称

暂定名称：**一起进步呀**

可选名称：

- 一起进步呀
- 恋爱成长日记
- 双人成长计划
- 小情侣监督站
- 今天也要变好呀

### 1.2 产品一句话介绍

**一款给情侣使用的共同成长小程序，两个人可以一起制定计划、每日打卡、互相提醒、互相监督，在可爱又温柔的氛围里一起变好。**

### 1.3 产品定位

这不是一个普通 Todo 工具，而是一个具有情侣陪伴感的成长小程序。

它解决的问题不是“我今天有什么任务”，而是：

> 我想努力，但一个人容易摆烂；如果有喜欢的人看见我、提醒我、鼓励我，我会更容易坚持下去。

### 1.4 产品关键词

- 情侣
- 共同成长
- 每日打卡
- 互相监督
- 温柔提醒
- 可爱陪伴
- 计划管理
- 成长记录

## 2. 产品背景

很多情侣之间会有共同进步的愿望，例如：

- 一起学习
- 一起运动
- 一起攒钱
- 一起早睡
- 一起读书
- 一起变自律
- 一起准备考试
- 一起变成更好的人

但现实中容易出现几个问题：

1. 计划说得很好，但没有持续记录。
2. 两个人不知道对方今天有没有完成。
3. 提醒对方容易显得像催促，缺少温柔表达。
4. 普通打卡软件缺少情侣之间的情绪价值。
5. 坚持几天后容易中断，没有重新开始的机制。

因此，本产品希望用一种可爱、轻松、温柔的方式，让情侣之间的计划监督变得不压迫、不严肃，而是像一种“互相陪伴的成长小游戏”。

---

## 3. 产品目标

### 3.1 MVP 阶段目标

第一版目标非常明确：

> 让两个人绑定情侣关系后，可以创建计划、查看彼此计划、每日打卡、互相提醒。

### 3.2 核心目标

- 让用户能快速绑定情侣关系。
- 让用户能创建个人计划和共同计划。
- 让双方可以互相看到计划进度。
- 让用户每天可以完成打卡。
- 让用户可以用可爱的方式提醒对方。
- 让首页能体现两个人共同进步的感觉。

### 3.3 暂不做的目标

MVP 第一版不做以下功能：

- 不做复杂社交广场。
- 不做多人小组。
- 不做复杂积分商城。
- 不做 AI 自动规划。
- 不做付费会员。
- 不做复杂数据分析。
- 不做过多个性化装扮。

第一版重点是：**先让你和女朋友真的能用起来。**

## 4. 目标用户

### 4.1 核心用户

情侣中的一方或双方，希望通过互相监督一起变好。

典型用户：

- 正在学习的情侣
- 想一起运动的情侣
- 想一起早睡早起的情侣
- 异地恋情侣
- 想共同存钱旅行的情侣
- 想培养长期习惯的情侣

### 4.2 用户画像

#### 用户 A：容易想很多但执行力不稳定的人

特点：

- 有上进心。
- 想学习、赚钱、成长。
- 经常制定计划。
- 容易三分钟热度。
- 需要外部监督和及时反馈。

需求：

- 希望有人看到自己的努力。
- 希望对方提醒自己不要摆烂。
- 希望计划能被记录下来。

#### 用户 B：温柔陪伴型伴侣

特点：

- 愿意鼓励对方。
- 不想用强硬方式催促。
- 希望两个人一起成长。
- 喜欢可爱、温暖、有仪式感的产品。

需求：

- 能看到对方今天有没有完成计划。
- 可以用温柔方式提醒对方。
- 可以记录两个人一起坚持的过程。

## 5. 用户核心场景

### 5.1 场景一：情侣绑定

用户打开小程序后，生成自己的邀请码，并发给女朋友。

女朋友输入邀请码后，两个人完成绑定。

绑定后首页显示：

> 你们已经绑定成功啦～  
> 从今天开始，一起变好吧！

### 5.2 场景二：创建个人计划

用户创建一个个人计划：

- 计划名称：每天学习 Java
- 每日任务：学习 1 小时
- 周期：30 天
- 提醒时间：晚上 9 点

创建后，对方可以在“TA 的计划”中看到该计划。

### 5.3 场景三：创建共同计划

两个人创建共同计划：

- 计划名称：每天一起运动
- 每日任务：运动 20 分钟
- 周期：21 天

共同计划需要两个人都打卡，才算当天完成。

### 5.4 场景四：每日打卡

用户完成任务后点击打卡，填写：

- 是否完成
- 今日总结
- 今日心情

V1.0 暂不支持图片上传；图片打卡放入 V1.1 后续版本。

打卡后，对方可以看到记录。

### 5.5 场景五：互相提醒

如果对方还没打卡，用户可以点击提醒。

提醒类型：

- 温柔提醒
- 认真监督
- 鼓励一下
- 夸夸对方

示例文案：

> 今天的小任务还没完成哦～要不要现在开始呀？

### 5.6 场景六：查看成长记录

用户可以查看：

- 连续打卡天数
- 本周完成率
- 两个人共同完成次数
- 历史打卡日历

页面氛围要像一本可爱的成长日记。

## 6. 产品核心功能

## 6.1 登录功能

### 功能说明

用户首次进入小程序时，通过微信授权登录，系统获取用户 openid，并创建用户档案。

### 页面元素

- 小程序 Logo
- 欢迎语
- 微信登录按钮
- 可爱插画区域

### 示例文案

```text
欢迎来到「一起进步呀」
和喜欢的人一起，把普通的每一天变成进步的一天。
```

### 功能规则

- 用户未登录时，只能访问登录页。
- 登录成功后进入首页。
- 如果未绑定情侣关系，首页引导用户去绑定。

---

## 6.2 情侣绑定功能

### 功能说明

用户可以生成邀请码，邀请另一半加入。

### 功能入口

- 首页未绑定状态
- 我的页面
- 情侣绑定页面

### 页面元素

- 我的邀请码
- 复制邀请码按钮
- 输入对方邀请码输入框
- 绑定按钮
- 绑定成功提示

### 业务规则

1. 每个用户只能绑定一个伴侣。
2. 邀请码唯一。
3. 用户不能绑定自己。
4. 已绑定用户不能重复绑定其他人。
5. 绑定成功后生成 couple_id。

### 可爱提示文案

```text
把这串神秘小代码发给 TA，
你们就可以一起开启成长空间啦～
```

绑定成功文案：

```text
绑定成功！
从今天开始，你们可以一起进步啦～
```

---

## 6.3 首页功能

### 功能说明

首页是用户每天最常看的页面，用于展示双方今日计划、打卡状态和共同成长进度。

### 首页模块

#### 模块一：情侣成长状态卡片

展示：

- 两个人昵称
- 绑定天数
- 共同坚持天数
- 今日完成状态

示例：

```text
你们已经一起进步 7 天啦～
今天也要继续加油呀！
```

#### 模块二：今日我的计划

展示当前用户今日需要完成的计划。

字段：

- 计划名称
- 今日状态
- 打卡按钮
- 连续天数

#### 模块三：今日 TA 的计划

展示伴侣今日计划。

字段：

- 计划名称
- TA 是否已完成
- 提醒按钮

#### 模块四：共同计划

展示两个人共同参与的计划。

字段：

- 计划名称
- 我的完成状态
- TA 的完成状态
- 今日共同完成状态

### 首页操作

- 点击计划进入详情。
- 点击打卡进入打卡页。
- 点击提醒发送提醒。
- 点击创建计划进入创建页。

## 6.4 创建计划功能

### 功能说明

用户可以创建个人计划或共同计划。

### 字段设计

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---:|---|
| 计划名称 | 输入框 | 是 | 如：每天背单词 |
| 计划类型 | 单选 | 是 | 个人计划 / 共同计划 |
| 目标说明 | 文本框 | 否 | 为什么要做这个计划 |
| 每日任务 | 文本框 | 是 | 每天具体做什么 |
| 开始日期 | 日期选择 | 是 | 默认今天 |
| 结束日期 | 日期选择 | 是 | 计划结束时间 |
| 提醒时间 | 时间选择 | 否 | 如：21:00 |
| 是否需要监督 | 开关 | 否 | 是否允许对方提醒 |

### 业务规则

1. 计划名称不能为空。
2. 结束日期不能早于开始日期。
3. 个人计划只需要创建者打卡。
4. 共同计划需要双方都可以打卡。
5. 计划创建后，绑定伴侣可见。

### 可爱提示文案

```text
把想变好的愿望写下来吧，
小小计划，也会慢慢发光～
```

创建成功文案：

```text
计划创建成功！
今天开始，向更好的自己靠近一点点～
```

## 6.6 打卡功能

### 功能说明

用户每日对计划进行打卡，记录完成情况。

### 打卡字段

| 字段 | 类型 | 是否必填 | 说明 |
|---|---|---:|---|
| 完成状态 | 单选 | 是 | 已完成 / 未完成 |
| 今日总结 | 文本框 | 否 | 今天做了什么 |
| 今日心情 | 单选 | 否 | 开心 / 一般 / 摆烂 / 超棒 |

V1.0 打卡只支持完成状态、今日总结、今日心情。图片打卡不进入 V1.0 实现范围，若 UI 中出现图片区域，只能作为后续版本占位，不实现真实上传逻辑。

### 业务规则

1. 同一个计划每天每个用户只能打卡一次。
2. 打卡后可以查看，但 MVP 阶段暂不支持修改。
3. 共同计划需要双方都打卡完成，今日共同状态才算完成。
4. 未完成也可以记录原因。

### 心情选项

- 开心
- 一般
- 有点摆烂
- 超额完成
- 需要抱抱

### 打卡成功文案

```text
叮！今日进步已记录～
你又比昨天更棒一点啦！
```

未完成记录文案：

```text
没关系，今天只是暂停一下。
明天我们再一起重新开始～
```

## 6.7 互相提醒功能

### 功能说明

用户可以给伴侣发送提醒消息，用温柔、鼓励或认真监督的方式提醒对方完成计划。

### 提醒类型

| 类型 | 说明 | 示例 |
|---|---|---|
| 温柔提醒 | 轻轻提醒对方 | 今天的小任务还没完成哦～ |
| 认真监督 | 稍微认真一点，但不命令、不制造压力 | 不许偷偷摆烂哦，要不要现在开始一点点？ |
| 鼓励一下 | 给对方动力 | 你已经坚持很久啦，再完成一天！ |
| 夸夸对方 | 表扬对方 | 今天的你也很努力，值得夸夸！ |

### 业务规则

1. 只能提醒已绑定的伴侣。
2. 可以针对某个计划提醒。
3. 提醒生成后进入对方提醒列表。
4. 对方查看后标记为已读。
5. MVP 阶段可先不接微信订阅消息，只做站内提醒。

### 可爱文案

```text
提醒已经飞过去啦～
希望 TA 收到你的心意，也愿意慢慢开始～
```

## 7. 页面结构

MVP 建议 4 个 Tab：

1. 首页
2. 计划
3. 提醒
4. 我的

- 打卡页不放在 TabBar 中，只能从首页、计划详情页、计划列表页进入。

## 10. 权限规则

### 10.1 用户权限

用户只能操作自己的数据：

- 修改自己的资料
- 创建自己的计划
- 给自己的计划打卡
- 查看自己的打卡记录

### 10.2 情侣可见权限

绑定伴侣后，双方可以互相查看：

- 对方计划
- 对方打卡状态
- 共同计划
- 共同成长记录

### 10.3 禁止操作

用户不能：

- 修改对方计划
- 删除对方计划
- 替对方打卡
- 绑定多个伴侣
- 查看非绑定用户数据

---

## 11. 核心业务规则

### 11.1 个人计划规则

- 创建者拥有编辑和结束权限。
- 伴侣只能查看、提醒、鼓励。
- 只有创建者可以打卡。

### 11.2 共同计划规则

- 双方都可以看到。
- 双方都可以打卡。
- 当天双方都完成，才算共同完成。
- 任意一方未完成，当天共同计划状态为未完全完成。

### 11.3 打卡规则

- 每个用户每天对同一个计划只能打卡一次。
- 打卡日期按当天自然日计算。
- MVP 阶段不允许补卡。
- 后续版本可以增加“补卡卡片”。

### 11.4 提醒规则

- 只能提醒绑定伴侣。
- 同一个计划一天可以提醒多次，但建议限制为 3 次，避免打扰。
- 提醒内容可以使用系统默认文案。

## 19. 产品风险

### 19.1 过度监督风险

如果提醒太频繁，可能让用户产生压力。

解决方案：

- 限制每日提醒次数。
- 文案偏温柔。
- 提供“今天不想被提醒”选项。

### 19.2 情侣关系变化风险

情侣绑定涉及关系状态变化。

解决方案：

- 后续版本支持解除绑定。
- 解除后双方不可继续查看彼此数据。

### 19.3 功能过重风险

如果第一版功能太多，会导致开发周期变长。

解决方案：

- 第一版只做核心闭环。
- AI、徽章、积分全部后置。

## 20. 最终总结

《一起进步呀》的核心价值是：

> 让情侣在日常生活中，用一种可爱、轻松、温柔的方式，互相看见彼此的努力，并一起变成更好的人。

第一版最重要的闭环是：

```text
绑定情侣关系 → 创建计划 → 查看彼此计划 → 每日打卡 → 互相提醒 → 看到成长记录
```

只要这个闭环跑通，这个小程序就已经具备真实使用价值。

第一版不要追求复杂，要追求：

- 能用
- 好看
- 温暖
- 你和女朋友真的愿意每天
