# DESIGN.md

## 产品视觉定位

「一起进步呀」是一款情侣共同成长、打卡、计划管理类微信小程序。整体视觉必须延续当前 UI 图的风格：可爱、温暖、情侣感、奶油粉色系、成长日记感。

设计关键词：

- 温柔陪伴：界面像一本两个人共同书写的成长日记。
- 轻甜情侣感：使用爱心、小熊、贴纸、柔和插画表达亲密感，但避免过度幼稚。
- 奶油粉色系：粉色作为主操作与情绪色，搭配奶油白背景。
- 大圆角卡片：所有主要内容以柔和卡片承载，边界清晰但不生硬。
- 清晰可读：真实手机上字体不能过小，按钮和点击区域必须易操作。

技术限定：

- 使用微信小程序原生开发。
- 页面必须使用 `WXML + WXSS + JS`。
- 不使用 React、Vue、Tailwind。
- 样式单位优先使用 `rpx`。
- 页面左右安全边距统一为 `32rpx`，对应设计要求的 `16px`。

文档边界：

- `PRD.md` 负责产品功能、页面结构、业务规则、开发优先级。
- `DESIGN.md` 负责视觉风格、颜色、字体、间距、圆角、按钮、卡片与组件规范。
- 如果视觉规范影响产品结构，例如 TabBar 页面数量，以 PRD 的产品结构为准。

页面结构统一标准：

- V1.0 页面总数为 10 个：登录页、首页、情侣绑定页、计划列表页、创建计划页、计划详情页、打卡页、提醒页、成长记录页、我的页。
- 底部 TabBar 固定 4 个：首页、计划、提醒、我的。
- 打卡页、成长记录页、创建计划页、情侣绑定页不放入 TabBar。

## 色彩规范

主色来自 UI 图底部色板，并按小程序使用场景整理如下。

### 品牌主色

| 用途 | 色值 | 使用说明 |
| --- | --- | --- |
| 主粉色 | `#FF8FAB` | 主按钮、当前 Tab、重要高亮、进度条、强调文字 |
| 浅粉背景 | `#FFE4EC` | 页面氛围背景、浅色标签、卡片内柔和区域 |
| 奶油白 | `#FFF8F1` | 页面主背景、卡片背景过渡 |
| 深文本色 | `#3D3D3D` | 页面标题、主要正文 |
| 橙黄色 | `#FFC6A8` | 提醒、温暖提示、轻量状态 |
| 淡紫色 | `#DCCBFF` | 次级分类、装饰标签 |
| 完成绿色 | `#8DD9A5` | 已完成状态、成功状态、共同计划完成反馈 |

### 推荐扩展色

```css
page {
  background: #FFF8F1;
}

.text-primary {
  color: #3D3D3D;
}

.text-secondary {
  color: #8E7F7A;
}

.text-muted {
  color: #B8AAA4;
}

.color-primary {
  color: #FF8FAB;
}

.bg-primary-soft {
  background: #FFE4EC;
}

.color-success {
  color: #8DD9A5;
}

.color-warning {
  color: #FFC6A8;
}
```

### 使用原则

- 主操作统一使用粉色 `#FF8FAB`。
- 完成状态统一使用绿色 `#8DD9A5`。
- 提醒、鼓励、轻提示使用橙色或黄色系 `#FFC6A8`。
- 页面不要出现大面积高饱和红色、蓝色、黑色。
- 背景以奶油白、浅粉、极浅渐变为主。
- 同一页面内颜色不宜超过 5 个主视觉色，避免杂乱。

## 字体大小规范

微信小程序中统一使用系统字体：

```css
page {
  font-family: -apple-system, BlinkMacSystemFont, "PingFang SC", "Helvetica Neue", Arial, sans-serif;
  color: #3D3D3D;
}
```

### 字号层级

| 类型 | rpx | px 参考 | 使用场景 |
| --- | ---: | ---: | --- |
| 页面大标题 | `44rpx` | 22px | 登录页品牌标题、重要数字 |
| 页面标题 | `34rpx` | 17px | 导航标题、模块标题 |
| 卡片标题 | `30rpx` | 15px | 计划名称、打卡标题 |
| 正文 | `28rpx` | 14px | 常规说明、列表内容 |
| 次要正文 | `26rpx` | 13px | 描述、时间、辅助信息 |
| 标签文字 | `24rpx` | 12px | 状态标签、按钮辅助文字 |
| 辅助小字 | `22rpx` | 11px | 字数统计、很轻的说明 |

### 字体使用原则

- 真实手机上正文不低于 `26rpx`。
- 主要可读文本推荐 `28rpx`。
- 不使用过小字体堆信息。
- 标题可适当加粗，正文保持自然字重。
- 重要数字可以使用 `44rpx-56rpx`，如“7 天”“85%”。

## 间距规范

页面左右边距统一：

```css
.page {
  padding-left: 32rpx;
  padding-right: 32rpx;
}
```

### 间距 Token

| 名称 | 值 | 使用场景 |
| --- | ---: | --- |
| `--space-xs` | `8rpx` | 图标与文字间距 |
| `--space-sm` | `12rpx` | 标签内部、紧凑元素 |
| `--space-md` | `16rpx` | 列表内部间距 |
| `--space-lg` | `24rpx` | 卡片内边距、模块间距 |
| `--space-xl` | `32rpx` | 页面左右边距、主要区块间距 |
| `--space-xxl` | `48rpx` | 页面大区块间距 |

### 布局间距

- 页面左右边距：`32rpx`。
- 卡片内边距：`24rpx-28rpx`。
- 卡片之间垂直间距：`20rpx-24rpx`。
- 列表项内部上下间距：`20rpx`。
- 标题与内容间距：`16rpx-24rpx`。
- 底部 TabBar 上方内容需预留 `120rpx` 以上，避免遮挡。

## 圆角规范

当前 UI 图的核心特征是大圆角和柔和边缘，圆角必须统一。

| 类型 | 圆角 |
| --- | ---: |
| 主卡片 | `40rpx` |
| 次级卡片 | `32rpx` |
| 输入框 | `20rpx-24rpx` |
| 主按钮 | `999rpx` |
| 标签 | `999rpx` |
| 弹窗 | `40rpx` |
| 图标底色块 | `20rpx-24rpx` |
| 头像 | `50%` 或 `999rpx` |

设计要求中的卡片圆角约 `20px`，在小程序中统一对应 `40rpx`。

## 阴影规范

阴影要柔和，像奶油色纸卡浮在背景上，避免强烈黑色投影。

### 主卡片阴影

```css
.card {
  box-shadow: 0 12rpx 32rpx rgba(255, 143, 171, 0.14);
}
```

### 轻量卡片阴影

```css
.card-light {
  box-shadow: 0 8rpx 24rpx rgba(61, 61, 61, 0.06);
}
```

### 悬浮按钮阴影

```css
.float-button {
  box-shadow: 0 10rpx 28rpx rgba(255, 143, 171, 0.35);
}
```

### 使用原则

- 阴影颜色优先使用粉色透明阴影。
- 不使用浓重黑色阴影。
- 卡片边框可搭配 `1rpx solid rgba(255, 143, 171, 0.12)`。
- 同一页面不要叠加过多阴影层级。

## 按钮规范

### 主按钮

用于登录、绑定、创建计划、完成打卡、保存等主操作。

```css
.btn-primary {
  height: 88rpx;
  min-height: 88rpx;
  border-radius: 999rpx;
  background: linear-gradient(180deg, #FF9BB4 0%, #FF6F96 100%);
  color: #FFFFFF;
  font-size: 28rpx;
  font-weight: 600;
  box-shadow: 0 10rpx 24rpx rgba(255, 143, 171, 0.28);
}
```

规则：

- 主按钮高度不能低于 `88rpx`，即 `44px`。
- 页面主按钮通常铺满内容宽度。
- 同一屏最多保留一个最强主按钮。
- 按钮文字使用 4-6 个字为佳，如“创建计划”“打卡完成”“绑定我们”。

### 次按钮

```css
.btn-secondary {
  height: 88rpx;
  border-radius: 999rpx;
  background: #FFFFFF;
  color: #3D3D3D;
  font-size: 28rpx;
  border: 1rpx solid rgba(61, 61, 61, 0.12);
}
```

### 小按钮

用于“发送”“查看详情”“已绑定”等轻量操作。

```css
.btn-small {
  height: 56rpx;
  padding: 0 24rpx;
  border-radius: 999rpx;
  font-size: 24rpx;
  background: #FF8FAB;
  color: #FFFFFF;
}
```

### 禁用按钮

```css
.btn-disabled {
  background: #E8E0DC;
  color: #B8AAA4;
  box-shadow: none;
}
```

## 卡片规范

卡片是页面的主要信息容器。

```css
.card {
  background: rgba(255, 255, 255, 0.86);
  border-radius: 40rpx;
  padding: 24rpx;
  border: 1rpx solid rgba(255, 143, 171, 0.12);
  box-shadow: 0 12rpx 32rpx rgba(255, 143, 171, 0.14);
}
```

### 卡片类型

- 首页顶部氛围卡：可使用浅粉渐变、小熊插画、爱心贴纸。
- 计划列表卡：左侧图标或头像，中间标题和描述，右侧状态或勾选。
- 数据卡：用于“已一起进步 7 天”“总完成率 85%”。
- 设置卡：用于“我的计划”“提醒设置”“打卡声音”等列表入口。
- 空状态卡：用于没有计划、没有记录时展示温柔提示。

### 卡片规则

- 卡片圆角统一约 `40rpx`。
- 卡片之间保持 `20rpx-24rpx` 间距。
- 卡片内图标使用浅色底，如浅粉、浅橙、浅紫、浅绿。
- 卡片内容要留白，不要压得太满。
- 不要使用直角卡片或强边框卡片。

### 输入框

```css

  background: #FFFFFF;
  border: 1rpx solid rgba(61, 61, 61, 0.08);
  padding: 0 24rpx;
  font-size: 26rpx;
  color: #3D3D3D;
}
```

### 多行输入

```css
.textarea {
  min-height: 144rpx;
  border-radius: 24rpx;
  background: #FFFFFF;
  border: 1rpx solid rgba(61, 61, 61, 0.08);
  padding: 20rpx 24rpx;
  font-size: 26rpx;
  line-height: 1.6;
}
```

### Placeholder

```css
.input-placeholder {
  color: #B8AAA4;
}
```

### 表单规则

- Label 使用 `26rpx-28rpx`，颜色 `#3D3D3D`。
- 输入框高度不低于 `80rpx`。
- 表单块之间间距 `28rpx-32rpx`。
- 字数统计放右侧，使用 `22rpx` 和浅灰色。
- 错误提示使用温和表达，如“这里还差一点点内容哦”。

## TabBar 规范

底部 TabBar 保持当前 UI 图风格：白色圆润、轻量图标、当前项粉色高亮。

### 推荐结构

TabBar 项：

- 首页
- 计划
- 提醒
- 我的

非 TabBar 页面入口：

- 打卡页只能从首页、计划详情页、计划列表页进入。
- 成长记录页只能从首页成长卡片或我的页进入。
- 创建计划页只能从首页或计划列表页进入。
- 情侣绑定页只能从首页未绑定状态或我的页进入。

### 样式规则

```css
.tabbar {
  height: 112rpx;
  background: rgba(255, 255, 255, 0.96);
  border-top: 1rpx solid rgba(61, 61, 61, 0.06);
  box-shadow: 0 -8rpx 24rpx rgba(61, 61, 61, 0.04);
}

.tabbar-item {
  font-size: 22rpx;
  color: #9C908C;
}

.tabbar-item.active {
  color: #FF8FAB;
}
```

### 图标规则

- 当前图标使用粉色 `#FF8FAB`。
- 未选中图标使用灰色 `#9C908C`。
- 图标尺寸建议 `44rpx-48rpx`。
- 文案尺寸 `22rpx`，不要再小。
- TabBar 不使用深色背景。
- TabBar 不出现“打卡”和“记录”入口，避免与 PRD 页面结构冲突。

## 空状态规范

空状态要温柔、轻盈，避免冷冰冰。

### 视觉组成

- 小熊、爱心、日历、清单等轻插画。
- 一句主提示。
- 一句辅助说明。
- 一个主操作按钮。

### 文案示例

没有计划：

- 主提示：`还没有新的小计划`
- 辅助说明：`一起定个小目标，把今天过得更认真一点。`
- 按钮：`创建计划`

没有打卡记录：

- 主提示：`这里还在等第一条记录`
- 辅助说明：`完成一次打卡后，成长足迹会出现在这里。`
- 按钮：`去打卡`

没有绑定情侣：

- 主提示：`还差一步，就能一起进步啦`
- 辅助说明：`输入对方的邀请码，开启你们的专属空间。`
- 按钮：`绑定我们`

### 样式规则

- 空状态卡片居中展示。
- 插画尺寸建议 `180rpx-240rpx`。
- 主提示 `30rpx`，辅助说明 `26rpx`。
- 不要使用夸张营销语或低龄化语气。

## 弹窗规范

弹窗用于确认完成、删除确认、提醒发送、绑定成功等。

### 样式

```css
.modal {
  width: 640rpx;
  border-radius: 40rpx;
  background: #FFFFFF;
  padding: 40rpx 32rpx 32rpx;
  box-shadow: 0 20rpx 60rpx rgba(61, 61, 61, 0.12);
}

.modal-title {
  font-size: 32rpx;
  font-weight: 600;
  color: #3D3D3D;
}

.modal-content {
  font-size: 26rpx;
  line-height: 1.6;
  color: #8E7F7A;
}
```

### 弹窗规则

- 弹窗圆角 `40rpx`。
- 内容居中或左对齐，按信息复杂度决定。
- 主按钮粉色，次按钮白底灰字。
- 删除类操作也不使用大面积刺眼红色，优先通过文案确认风险。
- 弹窗文案温柔直接，不绕弯。

文案示例：

- `今天也完成啦`
- `要把这个计划暂时放下吗？`
- `提醒已经送到对方那里啦`
- `你们的专属空间绑定成功`

## 页面布局规范

### 页面基础

```css
page {
  min-height: 100vh;
  background: linear-gradient(180deg, #FFF1F5 0%, #FFF8F1 42%, #FFFFFF 100%);
}

.page {
  min-height: 100vh;
  padding: 24rpx 32rpx 140rpx;
  box-sizing: border-box;
}
```

### 导航区

- 标题居中。
- 标题字号 `32rpx-34rpx`。
- 返回按钮保持系统风格或使用轻量线性图标。
- 页面顶部可适当留白，不要拥挤。

### 首页布局

- 顶部使用粉色氛围卡展示“已一起进步 X 天”。
- 中部展示今日计划，区分“我的计划”“TA 的计划”“共同计划”。
- 底部使用 TabBar。
- 首页应有温柔鼓励文案，但不要大段说明。

### 计划列表页

- 顶部使用分段 Tab：`我的计划 / TA 的计划 / 共同计划`。
- 列表项使用白色圆角卡片。
- 右下角可放粉色悬浮创建按钮。
- 计划状态用标签或圆形勾选表达。

### 创建计划页

- 表单分组清晰。
- 计划类型使用图标卡片选择。
- 开启提醒使用 switch，选中态粉色。
- 底部固定主按钮“创建计划”。

### 打卡页

- 顶部展示当前计划卡片。
- 完成状态使用粉色与绿色结合。
- 心情选择使用圆形表情按钮。
- 今日总结使用多行输入。
- V1.0 不实现图片上传，打卡页只包含完成状态、今日总结、今日心情。
- 如需保留图片区域，只能显示为“图片打卡将在后续版本开放”的占位提示，不提供上传入口和上传逻辑。
- 底部主按钮“打卡完成”。

### 提醒页

- 提醒页是 TabBar 页面，展示收到的提醒和发出的提醒。
- 提醒卡片使用浅粉、浅橙或白色卡片，按提醒类型区分轻微色彩。
- “认真监督”文案可以稍认真，但必须温柔，不使用命令式、压迫式表达。
- 空状态使用温柔提示，如“今天还没有提醒消息，也许你们都很自觉呢～”。

### 成长记录页

- 数据卡展示连续天数和完成率。
- 日历使用圆形日期点，高亮日期使用粉色。
- 时间线使用粉色线条和圆点。
- 记录文案简短，有日记感。

### 我的页

- 顶部个人信息卡使用头像、昵称、绑定状态。
- 设置列表使用白底圆角卡片。
- switch 选中态使用绿色或粉色，按功能语义决定。
- 页面整体保持轻量，不做复杂装饰。

## 文案语气规范

文案要温柔可爱，但不要幼稚。

### 语气原则

- 像亲近的人轻轻提醒，不像系统命令。
- 可以使用“呀、啦、哦”少量增加亲和感。
- 避免过度撒娇、卖萌、网络流行语。
- 鼓励表达要克制，不要制造压力。
- 情侣感要自然，不要油腻。

### 推荐文案

- `一起进步呀`
- `和你一起，成为更好的我们`
- `你们已经一起进步 7 天啦`

- `必须完成`
- `你失败了`
- `快点打卡`
- `不完成就落后了`
- `宝贝快来冲鸭`
- `绝绝子`
- `狠狠爱了`

## 需要避免的设计问题

- 不要重新发散成冷色科技风、极简商务风、深色风。
- 不要使用 React、Vue、Tailwind。
- 不要让页面左右边距不统一，统一使用 `32rpx`。
- 不要把正文做得太小，正文建议 `28rpx`，最低不低于 `26rpx`。
- 不要让主按钮低于 `88rpx` 高。
- 不要使用小圆角卡片，主卡片保持 `40rpx` 左右。
- 不要使用强烈黑色阴影或厚重边框。
- 不要大面积使用纯红色表达提醒。
- 不要在一个页面里堆太多爱心、贴纸、插画。
- 不要让卡片内部信息过密。
- 不要使用过多渐变，主按钮和顶部氛围卡可以用，普通卡片保持干净。
- 不要把情侣感做成低龄化或过度撒娇。
- 不要使用低对比度文字，浅色背景上的文字必须清晰。
- 不要让 TabBar 遮挡页面底部按钮或内容。
- 不要使用不可点击的小热区，按钮和主要点击区域高度建议不低于 `80rpx`。
- 不要让完成、提醒、主操作颜色混用：完成用绿色，主操作用粉色，提醒用橙色或黄色。
